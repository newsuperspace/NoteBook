Cocos2d-js 精灵Sprite的性能优化

2015-06-15 cocos2d-js,HTML5游戏开发,性能优化 COCOS2D-JS 1615次访问

    

    游戏是一种很耗费资源的应用，尤其是在移动设备中的游戏，性能优化是非常重要的。性能优化有很多方面，本章只介绍精灵相关的性能优化。

    精灵的性能优化可以使用精灵表和缓存：

    一. 使用纹理图集

    纹理图集(texture atlas)也成为精灵表（sprite sheet），它是把许多小的精灵图片组合到一张大图里面。使用纹理图集有以下优点：

    1. 减少文件读取次数，读取一张图片比读取一堆小文件要快。

    2. 减少渲染引擎的绘制调用并且加速渲染。

    3. Cocos2d-js全面支持Zwoptex和TexturePacker这两种工具，所以纹理图集制作简单。通过这两种工具可以生成纹理图集文件和纹理图集坐标文件plist。

    使用精灵表文件最简单的方式是使用图片资源路径和裁剪的矩形区域创建Sprite对象，其中创建矩形rect对象可以参考坐标文件plist数据参数，例如下面plist文件部分代码第三行：

    使用方式如下：

var sprite = new cc.Sprite("res.sheetPng",cc.rect(2,1472,488,580));

var texture = cc.textureCache.addImage("res.sheetPng");
var sprite = new cc.Sprite(texture, cc.rect(2,1472,488,580));

    二. 使用精灵帧缓存

    精灵帧缓存是缓存的一种。缓存有如下几种：

    1. 纹理缓存（TextureCache）。使用纹理缓存可以创建纹理对象，在上文我们已经用到了。

    2. 精灵帧缓存(SpriteFrameCache)。能够从精灵表中创建精灵帧缓存，然后再从精灵帧缓存中获得精灵对象，反复使用精灵对象时候，使用精灵帧缓存可以节省内存消耗。

    3. 动画缓存（AnimationCache）。动画缓存主要用于精灵动画，精灵动画中的每一帧是从动画缓存中获取的。

    这里我们主要介绍精灵帧缓存（SpriteFrameCache），要使用精灵帧缓存涉及到的类有：SpriteFrame和SpriteFrameCache。

    使用SpriteFrameCache创建精灵对象的主要代码如下：

1
2
cc.spriteFrameCache.addSpriteFrames(res.sheetPlist,res.sheetPng);                  //向精灵帧缓存添加精灵帧
var sprite = new cc.Sprite("#image.png");

    注意：精灵帧缓存不再使用后要移除相关精灵帧，否则如果再出现相同名称的精灵帧时，就会出现一些奇怪的现象。spriteFrameCache类中移除精灵帧的缓存函数如下：

    （1）cc.spriteFrameCache.removeSpriteFrameByName(name)；指定具体的精灵帧名移除

    （2）cc.spriteFrameCache.removeSpriteFrames()；    指定移除精灵缓存

    （3）cc.spriteFrameCache.removeSpriteFramesFromFile(url)；    指定具体的坐标文件移除精灵帧

    （4）cc.spriteFrameCache.removeSpriteFramesFromTexture(texture)；     指定纹理移除精灵帧 

    一般为了防止该场景中的精灵帧缓存影响下一个场景，可以在当前场景所在层的onExit函数中调用这些函数，相关代码如下：

1
2
3
4
 onExit:function(){
    this._super();
    cc.spriteFrameCache.removeSpriteFrames()；
 }

    onExit函数是层退出时回调的函数，与构造函数类似，都属于层的生命周期中的函数。
